    面试题准备

    1.HTML

    1.1 DOCTYPE的作用

    • 作用：声明文档类型，未声明会触发怪异模式-导致页面布局样式出现问题
    • 方式：<!DOCTYPE html>

    1.2 HTML语义化

    • 核心：用合适的标签表达内容含义（正确的标签做正确的事）
    • 好处：
      • 代码可读性高
      • 有利于SEO
      • 提升无障碍访问（方便屏幕阅读器解析）

    1.3 块级元素 vs 行内元素

    • 块级元素(div)：独占一行，可设置宽高，默认宽度100%
    • 行内元素(span)：不换行，不可设置宽高，宽高基于内容调整
    • 行内块元素：display: inline-block

    1.4 HTML5新特性

    • 语义化标签：header，footer，article
    • 本地存储：localStorage,sessionStorage
    • Canvas/WebGL：图形绘制
    • 新技术Web Worker，Web Socket(双向通信协议)

    1.5 meta 标签

    • 作用：定义网页元数据，控制编码、视口适配、SEO和浏览器行为。

    1.6 script标签async和defer区别

    • 无属性：立即执行，按顺序执行，阻塞html解析
    • async：谁先加载完谁先执行 - 适合独立脚本，如统计代码，可能阻塞html解析
    • defer：按顺序执行 - 适合依赖 DOM 的脚本，不阻塞html解析（html解析完顺序执行）

    1.7 Cookie vs storage

    • 特点：
      • Cookie：4KB，可设置过期时间，无设置的话默认浏览器关闭失效。请求自动带。
      • LocalStorage：5-10M，永久保存，需手动清除。
      • SessionStorage：5-10M，标签页关闭失效。
    • 区别：
      • Cookie,localStorage：在所有同源的窗口下可以共享，SessionStorage不同浏览器窗口不会进行共享。
    • 应用：
      • Cookie(token)
      • localStoragea:用户主题设置
      • sessionStorage:表单草稿
    • 正常项目中为什么选择localStorage存储token？
      • 规避 CSRF（跨站请求伪造）：Token不随请求自动发送会天然降低CSRF风险
      • 跨域友好：避免Cookie的跨域配置复杂度
      • 前后端分离架构：前端需手动将Token加入请求头（如 Authorization），localStorage 更易控制

    1.8 浏览器渲染 HTML 的过程

    • 过程
      • 解析 HTML → 构建 DOM 树
      • 解析 CSS → 构建 CSSOM 树
      • 合并 DOM + CSSOM → 生成渲染树
      • 布局（Layout） → 绘制（Paint）

    1.9 SEO优化

    • 语义化标签
    • 图片优化：img标签添加alt属性
    • URL友好：使用短横线分隔（/details）

    2.CSS

    2.1 盒模型

    • 标准盒模型：元素width/height = content的宽高
    • ie盒模型：元素width/height = content + padding + border
    • 切换：box-sizing: content-box(默认) 或 border-box（推荐使用）

    2.2 BFC-块级格式化上下文

    • 定义：一种独立的布局环境，内外互不干扰（玻璃罩）
    • 触发方式：
      • float: left/right; /* 浮动 */
      • overflow: hidden/auto; /* 溢出隐藏 */
      • position: absolute/fixed; /* 绝对定位 */
      • display: inline-block/flex/grid/table-cell; /* 特殊display值 */
    • 作用：
      • 外边距重叠 - BFC隔离
      • 清除浮动-浮动元素高度塌陷 - BFC包含
      • 阻止文字环绕 - BFC阻止
    • 示例：

    2.3 水平垂直居中的方式

    • flex布局
    • 绝对定位 + Transform
    • Grid

    2.4 flex和Grid的区别

    • flex: 一维布局
      • flex: 1 => ==> flex:1 1 auto; ==> flex-grow:1;flex-shrik1;flex-base:auto;本身大小
    • grid：二维布局 - 解决复杂网格布局

    2.5 回流和重绘

    • 定义：
      • 回流：布局变化触发重新计算（修改元素尺寸）
      • 重绘：样式变化但布局不变化（修改颜色）
      • 回流一定会触发重绘，重绘不一定会回流
    • 回流重绘影响浏览器性能，优化方式：
      • 避免重复操作dom
      • 使用transform/opacity触发GPU加速（减少重排）// 浏览器通过GPU处理这两个属性，而不是重新计算整个页面的布局

    2.6 伪类vs伪元素

    • 概念
      • 伪类(单冒号)：描述元素状态。如：:hover，:first-child等
      • 伪元素(双冒号)：创建虚拟元素。如：::after清除浮动等

    2.7 postion定位

    • 分类
      • static:默认值，没有定位
      • relative:(相对定位)相对于元素原来位置进行定位
      • absolute:(绝对定位)相对于最近的有定位的父元素进行定位，脱离了文档流
      • fixed:(固定定位)相对于浏览器窗口定位，脱离了文档流
      • sticky:(粘性定位)relative和fixed的结合，根据滚动的位置进行定位

    2.8 css选择器优先级

    !important > 行内 > ID > 类 > 元素

    2.9 css预处理器有哪些

    • 定义：通过编程方式增强CSS的工具，需编译为原生CSS
    • 核心优势：变量，嵌套，混合（Mixins）-代码复用，模块化（@import）
    • 分类
      • less
      • sass - 无花括号 - scss
    • less和sass比较
      • sass：更强大，支持循环、条件语句

    2.10 固定单位和相对单位

    • 区分：
      • px：固定单位，像素值（精确控制元素的尺寸 - 按钮，图标）
      • rpx：响应式像素，根据屏幕宽度（750rpx）自动适配（微信小程序专用单位）
      • em：相对单位，基于当前元素或父元素的字体大小，继承链会影响值
      • rem：相对单位，基于根元素的字体大小
      • vw,vh：基于视口大小，适合响应式设计（全屏背景图）
      • %：依赖父元素，属性不同基准不同

    2.11 响应式布局的实现方式

    • 方式：
      • 媒体查询 - 针对不同的屏幕尺寸设置不同的样式
      • Flexbox/Grid - 内容自适应排列
      • 相对单位（vw/rem） - 整体比例缩放
    • 示例

    2.12 css3新特性

    • 分类
      • 盒模型
      • 动画
      • 弹性布局
      • 阴影
      • 动画
      • 选择器（:first-child）

    2.13 css动画

    • transition：过渡效果，需要触发，单次动画
    • animation：复杂动画，自动或触发均可，可循环，暂停

    3. Javascript

    3.1 Js数据类型及判断方法

    • 分类
      • 基本数据类型：存放在栈内存中
        number,string,boolean,undefined,null,symbol,bigInt
      • 引用数据类型：存放在堆内存中
        object,array,function
    • 判断数据的方式
      • typeof：无法判断null和具体的引用数据类型
      • instanceof：检测对象是否是某个构造函数的实例，依赖于对象的原型链
      • constructor：查看对象对应的构造函数，除了undefined和null其他类型都能判断
      • Object.prototype.toString.call：万能
    • 判断数组的方式
      • [] instance of Array
      • [].constructor === Array
      • Object.prototype.toString.call([])===’[object Array]’
      • Array.isArray([]) //true
    • Symbol：独一无二的值（解决命名冲突问题）
    • bigInt：大整形–更大数值运算

    3.2 包装类

    • 概念：基本数据类型无法调用任何属性和方法，但通过包装类可以调用属性和方法。

    3.3 js函数类型

    • 函数类型
      • 1.普通函数 function Test(){ return arguments }
      • 2.立即执行函数 let num = (function () { return 8 + 9}())
      • 3.箭头函数 let num = (a, b, …rest) => { return … }
    • 箭头函数和普通函数区别
      • 1.普通函数可以作为构造函数，箭头函数new会报错
      • 2.普通函数有自己的原型对象，箭头函数没有
      • 3.普通函数有arguments(实参列表)，箭头函数有rest(获取函数多余参数)
      • 4.普通函数this指向undefined，箭头函数指向上一级作用域的this指向

    3.4 构造函数

    • 概念：用于创建对象的函数
    • new一个对象的过程
      • 1.var obj = {};
      • 2.obj.proto = Person.prototype; // obj 继承Person.prototype上的属性和方法
        obj.constructor = Person;
      • 3.Person.call(obj) // 让函数的this指向这个对象
      • 4.返回obj对象

    3.5 原型和原型链

    • 原型：每个函数都有一个prototype属性，这个属性指向一个对象，这个对象被称为该函数的原型对象。
    • 原型作用：存放构造函数构造出对象的公共的属性和方法(构造函数.prototype)
    • 原型链：Js实现继承的一种方式(自身 -> 原型对象 -> 原型对象的原型 -> null)
    • constructor：每个原型对象都有个constructor属性，该属性指向该原型对象的构造函数

    3.6 Js继承方式

    • 概念：一个对象通过某种方式获取另一个对象的属性和方法
    • 方式
      • (1)通过原型链继承 - 会继承不必要的属性和方法
      • (2)通过构造函数继承
      • (3)通过公用原型
      • (4)寄生组合式继承（圣杯模式） - 最优解
      • (5)ES6 Class继承 - 底层基于寄生组合式继承
    • 应用场景
      • (1)框架组件扩展
      • (2)公共逻辑复用
      • (3)UI 组件库开发
      • (4)插件系统设计
    • 现代开发建议
      • 优先使用 ES6 Class 语法
    • 代码实现

    • 区别：
      • call,apply改变this指向后便执行该函数,bind返回一个新函数必须手动调用
      • call和bind一个参数一个参数传，apply以数组的方式传
    • 示例：

    3.8 闭包

    • 概念：函数里面嵌套函数，并把内部函数返回出来，内部函数可访问外部作用域的变量
    • 作用：
      • 封装私有变量 – 继承的圣杯模式
      • 模块化开发，防止污染全局变量
      • 高阶函数（节流和防抖）
    • 内存泄漏：无用的内存还在占用，得不到释放和归还
      • 闭包
      • 被遗弃的定时器
      • dom元素清空还在引用
    • 内存溢出：
      • 内存加载数据过大
      • 代码死循环

    3.9 创建对象方式

      • 2.new Object() - 等价于对象字面量
      • 3.构造函数（可复用，实例共享原型方法）
      • 4.Object.create - 创建中间对象隔离父子类原型，避免副作用（继承且隔离）

    3.13 js类似方法比较

    • for循环和forEach哪个比较快
      • for>forEach 没有额外的函数调用栈和上下文
      • forEach不能使用break或return跳出循环可使用try catch
    • map和forEach有什么区别
      • forEach不会返回数据，map会返回一个新的数组
    • for in 和for of区别
      • for in 遍历的是索引 for of 遍历的是值
      • for in 会遍历对象的整个原型链(可用hasOwnproperty区分)

    3.15 深浅拷贝

    • 概念：
      • 浅拷贝：浅拷贝:只拷贝一层，拷贝第一层基本数据类型的值及引用数据类型的地址
      • 深拷贝：拷贝多层

    3.16 var let const区别

    • 区别
      • 作用域：var 函数作用域，let/const 块级作用域
      • 变量提升：var 存在变量提升，let/const 存在暂时性死区，声明前访问会报错
      • 赋值：var/let 可重复赋值，cosnt不能重新赋值（引用数据可修改内部属性）
      • 重复声明：var 允许重复声明，let/const 同一作用域禁止重复声明
    • 总结：优先用 const，需修改时用 let，避免使用 var
    • 相关概念：
      • 作用域：规定变量和函数的可使用范围
      • 作用域链：查找变量和函数时，从局部到全局依次查找
      • js执行：语法分析，预编译(创建当前环境的执行上下文)，解释执行
      • 预编译：全局预编译和函数预编译

    3.17 事件循环

    • 概念：不断执行宏任务和当前宏任务包含的所有微任务的循环过程
      • 宏任务：script全部代码,setTimeout,setInterval,setImmediate,I/o
      • 微任务：Prcess.nextTick,promise.then(new Promise过程属于同步任务，resove或reject后才算微任务)
    • 作用：js的事件循环模型保证了代码的异步执行
    • 过程：
      • 同步任务立即执行
      • 异步任务进入任务队列（异步任务分为宏任务和微任务）
      • 微任务优于宏任务执行
    • 示例

    3.18 this指向及改变this指向

    • 分类
      • 普通函数：严格模式指向undefined，否则指向window
      • 对象中函数：指向对象
      • 构造函数，this指向新创建的实例对象
      • 箭头函数：继承外层作用域的this
    • 修改this指向：call,apply,bind

    3.19 事件委托

    • 相关概念
      • 事件捕获：根元素 -> 目标元素 (需要手动触发)
      • 事件冒泡：目标元素 -> 根元素
    • 原理：利用冒泡机制在父元素上集中处理子元素的事件
    • 优点：减少事件绑定，节省内存占用
    • 示例：商品列表，用户点击某个商品查看详情

    3.20 return break continue 区别

    • 区分
      • return：用于返回函数的结果并终止函数的执行（无论嵌套多深，只在函数中使用）
      • break：只能跳出当前所在的循环（只能跳出一层）
      • continue：中断本次循环，继续下次循环

    3.21 promise

    • 作用：异步编程的解决方案，支持链式调用，可以解决回调地狱问题
    • 异步编程：
      • fs文件操作，
      • 数据库操作
      • ajax
      • 定时器
    • 异步编程解决方案：
      • 回调函数（回调地狱 - 嵌套得过深，导致代码结构复杂、难以维护）
      • promise
      • generator
      • async await

    3.22 虚拟列表怎么实现的

    • 场景：适用于渲染大量数据时，尤其是当数据量大到无法一次性渲染到页面时，渲染数据量大，浏览器卡顿
    • 概念：动态渲染可视区域元素，优化长列表性能
    • 实现技术点：
      • 监听滚动事件
      • 动态计算可见区域（计算起始索引），添加缓冲区域
      • 动态高度支持 - 使用ResizeObserver监听每个列表项的实际高度
      • 性能优化 - 使用CSS transform实现位置偏移
        • GPU加速：现代浏览器会触发硬件加速
        • 避免回流：只触发重绘（Repaint），性能更好
        • 批量处理：只需修改单个属性即可移动整个可视区域

    3.23 手写节流，防抖

    • 概念：
      • Throttle节流：多次执行变成每隔一段时间执行（滚动事件，窗口大小调整，拖拽操作，按钮防重复点击）
      • Debounce防抖：多次执行只执行最后一次（输入框实时搜索）

    1.解决跨域方式

    • CORS（跨域资源共享）：后端设置响应头，允许指定域名的请求。（后端明确告诉浏览器允许跨域）
    • 代理服务器：前端通过同域的中间服务器（如Vite开发服务器、Nginx）转发请求，避开浏览器限制
    • JSONP：利用<script>标签无跨域限制的特性(仅限GET请求，已过时)

    2.浏览器缓存机制

    • 概念
      • 强缓存：直接读本地，不联系服务器（看过期时间）
      • 协商缓存：缓存过期后，问服务器「能用旧的吗？」
    • 应用：
      • 问题1：前端框架生成的静态资源可以通过强缓存来提高性能：强缓存 + 哈希文件名 - 控制缓存（内容变→文件名变→缓存失效）
      • 问题2：HTML被强缓存 → 用户看不到页面更新。 HTML文件：禁用强缓存（Cache-Control: no-cache），用协商缓存确保及时更新
      • 对于不频繁变化的API数据，可以利用协商缓存来减少请求量

    3.从输入URL到页面加载的过程

    • 1.URL解析：判断地址是否错误
    • 2.DNS解析：解析域名->ip地址
    • 3.建立tcp连接(tpc三次握手)
    • 4.发送http(https)请求
    • 5.服务器响应
    • 6.浏览器接受响应并解析渲染页面
    • 7.断开tcp连接
